# RustEngine 后续开发计划 (2026-06-16)

## 项目现状

**17 个 crate**, ~84K 行源码。所有高优先级和中优先级任务已完成。低优先级任务 #12 (插件系统), #13 (Mod 系统), #15 (文档大修) 已完成。

**剩余低优先级任务 (大后期):**
- #11 Android 目标 — 需要 Android NDK
- #14 VR/AR 支持 — 需要 OpenXR

## 新增任务

### 高优先级 🔴

#### 1. 修复已知测试失败
- `engine-asset` 测试失败 — 缺少 `tempfile` dev-dep
- `engine-core` 示例使用过时的 `KeyCode` 变体
- **涉及**: `engine-asset`, `engine-core`
- **成功标准**: `cargo test --all` 全部通过

#### 2. 添加更多示例游戏
- 2D 平台跳跃示例 (精灵、物理、输入、瓦片地图)
- 3D FPS 示例 (网格渲染、物理、摄像机、音频)
- 多人游戏示例 (网络、权威、预测)
- **涉及**: `engine-core/examples/`
- **成功标准**: 每个示例可运行, 有详细注释

#### 3. 性能基准测试
- ECS 迭代基准 (1K, 10K, 100K 实体)
- 物理步进基准 (100, 1000, 10000 刚体)
- 渲染提交基准 (100, 500, 1000 draw call)
- **涉及**: `engine-ecs`, `engine-physics`, `engine-render`
- **成功标准**: 发布基准数据, 无性能退化

### 中优先级 🟡

#### 4. 代码质量改进
- 修复所有 clippy 警告
- 添加缺失的文档注释
- 统一错误处理模式
- **涉及**: 全 crate
- **成功标准**: `cargo clippy --all` 无警告

#### 5. 集成测试
- 跨 crate 工作流测试
- 场景加载 → 渲染 → 验证
- 物理模拟 → 碰撞检测 → 验证
- **涉及**: 全 crate
- **成功标准**: ≥20 个端到端测试

#### 6. CI/CD 改进
- 添加 macOS CI
- 添加 WASM CI
- 添加 Android CI (如果 NDK 可用)
- **涉及**: `.github/workflows/`
- **成功标准**: 所有平台 CI 通过

### 低优先级 🟢

#### 7. 文档改进
- API 参考生成 (cargo doc + 自定义 CSS)
- 架构指南 (渲染管线图、ECS 设计)
- 从其他引擎迁移指南 (Unity/Godot/Bevy)
- **涉及**: `docs/`
- **成功标准**: 新用户可按照文档从零构建完整游戏

#### 8. 示例游戏套件
- 完整的俄罗斯方块游戏
- 完整的平台跳跃游戏
- 完整的 3D FPS 游戏
- **涉及**: `examples/`
- **成功标准**: 每个游戏可运行, 有详细注释
